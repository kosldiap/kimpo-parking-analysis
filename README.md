# 김포공항 주차수요 분석 · 예측 (2016~2018)

김포국제공항 6개 주차장의 **주차관제시스템 입·출차 전수 데이터(563만 건)** 를 정제·분석하고,
**시간당 주차수요 예측 모형**을 구축한 프로젝트. 최종 목표는 교통센터(주차빌딩) 신축사업의
**장래 수요·편익 분석**(시나리오)이다.

> ⚠️ **원본 데이터는 저장소에 포함되지 않습니다.** raw(.xls) 약 2.5GB + 개인정보 포함 → `.gitignore` 제외. **코드만** 버전 관리.

## 주요 인사이트

1. **국내선 만성 초과수요** — 최대 점유 **122%**(정원 4,052대, 주차대행 overflow), 만차일수(공식) 국내1P 연 **365일**, 입차 탐색시간 2014년 2:40→2015년 5:26. → **신축 정당성의 정량 근거.**
2. **수요 동인이 주차장별로 완전히 다름** — 국내선 새벽5시 피크(여객), 화물 업무시간 쌍봉(물류). → 통합이 아닌 **유형별 개별 모형**이 타당.
3. **"여객→주차" 관계가 공식자료 대비 98% 일치** — 여객 예측 기반 장래수요 추정의 신뢰 기반.
4. **주차가 항공 출발을 2시간 선행**(교차상관 검증) — 수속·보안·대기·쇼핑 리드타임.

## 모델 설명 — 왜 XGBoost인가

네 가지 모델을 같은 데이터로 비교했다. **우리 실제 2018 holdout R²(운영형)** 기준:

| 모델 | 개념 | 운영형 R²(국내/국제/화물/직원) |
|---|---|---|
| **Linear** | Y=a₁X₁+…+b 직선 관계. 빠르고 해석 쉬움 | 0.90/0.90/0.89/**0.58** |
| **RandomForest** | 의사결정나무 다수의 평균. 비선형·상호작용 자동 학습, 변화에 다소 둔감 | 0.92/0.91/0.94/0.60 |
| **XGBoost** | 이전 나무의 오차를 다음 나무가 보정(부스팅). 비선형+최근변화에 가장 민감 | **0.93/0.91/0.94**/0.54 |
| **Prophet** | 추세+계절성+휴일 분해. 시계열 특화, 장기예측 가능 | 0.73/0.54/0.78/0.20 |

**순서: XGBoost ≳ RandomForest > Linear ≫ Prophet.** 이유는 주차수요의 특성과 맞물린다:
- 주차수요는 **시간대별로 극단적 변동**(새벽 피크) + **비선형·상호작용**(우천×야간 등) → 트리계열(XGB/RF)이 선형보다 유리.
- **최근 1~2시간/전일·전주(lag)가 가장 강력한 신호** → 부스팅으로 lag를 가장 잘 활용하는 XGBoost가 최상위.
- Linear도 lag 덕에 운영형에선 0.90까지 나오지만(단순해서 상한), lag를 뺀 **구조형에선 0.44~0.74로 급락**.
- **Prophet은 항공·기상을 회귀변수로 받았음에도 열위** — 핵심은 **자기상관(lag)을 쓰지 않아** 단기 변동을 못 잡기 때문(시계열 분해 특화의 한계).

> ⚠️ **두 체제 구분이 핵심**: lag를 쓰는 운영형은 단기예측엔 최강이나 **미래 시나리오엔 lag가 없어 못 씀**. 신규시설 장래수요(#7)에는 **구조형(no-lag, R²0.84~0.89)** 을 쓴다. (그림 `12`,`13`)

## 추가 인사이트 (운영·설계 기준, `src/extra_insights.py`)

**① 점유율 부하지속곡선 + 설계기준** — 국내선만 정원 부족, 정량화:
| 유형 | 만차(≥100%) 연 시간 | 설계기준 점유(연 30위) |
|---|---|---|
| **국내선** | **1,018h (≈42일)** | **115%** |
| 국제선 | 0h | 89% |
| 화물 | 0h | 40% |
| 직원 | 0h | 4% |
→ **국내선만 구조적 부족**(신축 근거), 나머지는 여유. 신축 규모 ≈ 국내선 정원의 +15% 수준.

![부하지속곡선](reports/figures/17_load_duration.png)

**② 회전율(1면당 일 회전)** — 화물 1.15(회전형) ≫ 국제선 0.78 > 국내선 0.57(점유형) ≫ 직원 0.07.
→ 유형별 설계 차별화. 점유형(국내·국제)이 카셰어링/AV에 더 취약.

**③ 게이트 부하** — 국내선 입차피크는 5시지만 **입+출차 합피크는 18시**(입차↓·출차↑ 교차) → 게이트·차로 설계는 합피크 기준.

**④ 매출·감면** — 국내선 요금 623억/수입 529억, **감면율 15.1%**(화물 21%·직원 5%). 건당수입 국내선 20,949원(장기주차) vs 국제선 10,606원.

**⑤ 명절·연휴 배율(평일=1)** — 여객(국내·국제)은 휴일에도 **0.85~1.0(거의 무관)**, 화물·직원은 **0.23~0.33(급감)**. → 여객주차는 상시 수요, 화물·직원은 업무일 패턴.

## 파이프라인 (`src/`)

| 모듈 | 역할 |
|---|---|
| `config.py` | 경로, 주차장 정규화, 4유형 매핑, 실제 정원, 상수 |
| `etl.py` | .xls 공통 리더 (포맷차이·멀티시트·날짜혼재 흡수) |
| `aggregate.py` | 전 파일→트랜잭션 마스터 + 일/시간 집계 (중복·구간·요금0 처리) |
| `eda.py` | 탐색 시각화 (시간/요일/계절/체류/상관) |
| `occupancy.py` | 입·출차 누적으로 동시주차(점유율%) 복원 |
| `features.py` | 4유형 시간당 특성행렬 (달력·공휴일·연휴·lag·외부결합) |
| `fetch_flights*.py` / `fetch_weather.py` | 외부데이터 수집 |
| `lead_analysis.py` | 주차→출발 선행시간 교차상관 추정 |
| `model_ml.py` | Linear/RF/XGBoost (운영형·구조형) |
| `model_ts.py` | Prophet (시계열 비교군) |
| `evaluate.py` / `evaluate_compare.py` | 지표(MAE/RMSE/MAPE/WAPE/R²) + 통합비교·교차검증 |
| `balking_analysis.py` | 점유율↔비주차회차 관계 |
| `validate_official.py` | 공식 월별 대비 재구성 검증 |

**실행 순서**: `aggregate.py` → `occupancy.py` → (`fetch_*`) → `features.py` → `model_ml.py [structural]` → `model_ts.py` → `evaluate_compare.py`
